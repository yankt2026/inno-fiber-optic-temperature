# Content Guide

## Editorial scope

Write only about point-based fluorescent fiber optic temperature sensing, probes, measurement instruments, OEM modules, and their transformer, reactor, switchgear, laboratory, high-voltage, EMI and RFI applications.

## Product claims

Use product names and technical values only after confirming them on `https://www.fjinno.net/Products/` or a relevant linked product page. Never transfer a value between models. If a value is not confirmed, use one of the approved qualifications in `src/data/products.ts`.

## Adding a product

Add a complete entry to `src/data/products.ts`. The route is generated automatically. Set `image` only after copying a clearly matching local file into `public/images/products/` and recording it in both image inventories.

## Adding an article

Add the topic to `src/data/pages.ts` for the current data-driven resource template, or create a validated Markdown entry in `src/content/resources/` when moving an article to the Content Collection workflow. Assign one primary intent, link to a confirmed product, a relevant solution and two related resources.

## Style

Answer the query early. Use short technical paragraphs, descriptive H2 headings, tables where they clarify selection, explicit limitations, and a natural engineering consultation CTA. Do not add dates, invented citations, unsupported results, customer stories or AI disclosures.
